# Canonical LinodeMCP tool capability tags.
# One "<tool>	<Capability>" per line, sorted by tool name. Capability is one
# of: Read, Write, Destroy, Admin, Meta. This file pins the capability tier of
# every tool so the Go and Python registries agree; a tool exposed under a
# different capability in one language would land it in a different profile.
# Admin = endpoints the Linode API gates on the account/child_account superscope
# (account administration, profile token/TFA/security/phone self-service, the
# Managed surface). Resource-scope ops (databases, object_storage, images,
# firewall, longview, events, support) are never Admin even when sensitive;
# secrets are handled by audit redaction, not the capability tier.
# Parity tests enforce this file in both languages:
#   go/internal/server/tools_capabilities_test.go
#   python/tests/unit/test_tools_capabilities.py
# When adding, renaming, retiering, or removing a tool, update this file and both
# implementations together or the gates fail.
